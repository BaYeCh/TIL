# 멘토링(최종원 멘토)

## 프로그래밍을 잘 하려면

- 코더가 아닌 프로그래머가 되자
- 잘 짜여진 다른 사람의 코드를 보고 이해하고 따라하자
- 검색은 Google
- Stack Overflow를 활용
  - 오류에 대한 정보를 얻어갈 수 있음
- 한 분야에(Front End or Back End or DB) 집중하는 것이 좋다
- 현 실력이 부족하다고 생각이 되면 온라인 과정을 병행하는 것도 고려해볼 만함
- 멀캠에서 배운 내용을 쓰지 않을 수도 있음, 회사와의 커뮤니케이션에 집중
- 목표하는 회사가 있다면 그 회사의 스펙에 대해서 면밀히 분석하고 준비하라.
- 코딩테스는 반드시 준비할 것. 정답을 맞추는 것 또한 중요하지만 변수를 설정할 때 의미가 확실하게 부여하는 것과 같은 가독성이 쉬운 코드를 작성하는 것에 목적을 가져라.

## 빅데이터

- 개발자로부터 시작이 기획의 초석이 될 수 있다
- 4V
- 휴먼 빅데이터
  - 사람이 만들어 놓은 시스템에서 생성되고 저장된 데이터
- 시스템 빅데이터
  - 기계, 서버로부터 발생된 데이터
- 스포츠와 빅데이터를 연결하여 프로젝트가 성행 중
- 소셜 빅데이터
  - 흔히 접하는 빅데이터(뉴스, 소셜미디어, 이미지)
- 빅데이터의 구분은 쓸모 없음
- 소셜 빅데이터와 텍스트 빅데이터를 활용한 서비스가 급증
- 소셜 자체만으론 할 수 있는 영역이 줄어져 타 분야와 융합을 통해 인사이트를 도출/분석함
- 교차분석의 방법으로 빅데이터 분석
- 텍스트 빅데이터 분석이 다시 뜰까?
  - 최근 AI스피커의 활용과 독거노인 관리를 위한 대화분석으로 텍스트 빅데이터 분석의 수요가 급증
  - 거의 모든 공공기관/ 금융기관의 챗봇 도입
  - 2020년 국회의원 선거 민심 분석
  - 대선 예측

# 취업 관련

- 자신이라는 상품을 기업에게 ''설득''
- 기업을 정하고 그에 맞추어 자기소개서를 작성하는 것이 전략적
- 면접
  - 나의 서류에서 무엇이 그들에게서 관심을 끌었는가?
  - 해당기업에 대한 업무 분석이 꼭 필요하다.
  - 대략적인 기업과 직무를 설정해야 함
  - 면접 질문을 찾기
    - 잡플래닛
    - 잡코리아

  - 현직자와 커넥션을 갖도록 노력하라.
  - 경력 단절 기간이나 취준기간 동안의 "공백"에 관한 질문
    - 삶의 태도 : "가치관"
    - 그 동안 무엇을 했는가? : "무엇"
    - 눈에 잘 보인다! : 이력서는 보기 쉬움, 숫자는 눈에 잘 보임
    - 공백 기간 동안 지원 직무를 위해서 무엇을 하였나요?
      - 한 것이 있음
        - 인턴, 외부 교육, 아르바이트

      - 한 것이 없음 (직접적으로 연결되는 것 중에서)
        - 솔직히 인정
        - 추후에 구체적인 노력에 대해서 언급할 것
        - 유사-간접 경험을 바탕으로 언급

  - 자기 소개
    - 왜 물어 보는가?
      - 우리 회사에 대한 진정성
        - 지원 기업에 필요한 직무 역량은 무엇인가?

      - 면접 시 질문 베이스
        - 자기소개를 잘 하면 꼬리 질문이 계속 나옴

      - 이력서, 자소서 등을 읽을 시간을 확보하기 위해서
    - 스킬
      - 침묵(2~3초 가량)을 적절히 활용
      - 핵심 메시지

    - '저는 지원하는 회사에 쓸만한 사람입니다'
      - 무엇을 어필하나?
        - 직무 역량 (지식, 기술, 태도) : 굳이 여기서?
        - 뽑아줄 이유를 주어야 하기 때문에
        - 태도보다는 지식과 기술에 집중

    - 왜 우리 회사에 지원했는가?
      - 관점 변화
        - 기업이 나를 찾는 이유
          - 나의 강점과 연결
          - 상호 성장이 가능
      - 기업 분석이 중요!
        - 지원 직무의 지원 기업의 강점, 이슈 등을 파악
        - 개발 성공사례
        - 직무 현직자(혹은 CEO) 인터뷰 확인
        - 기업의 홈페이지의 홍보/PR/보도자료
        - CEO 인사말
    - 마지막으로 하고 싶은 말
      - 어설프게 길게 할거면 차라리 안하는게 낫다.
      - 면접의 결과를 뒤집기
        - 내가 충분하게 나를 보여주지 못함
        - 하지 못한 준비된 답변이 있음
        - 이거 하지 않으면 후회할 것

  - 1분 자기소개
    - 직무 역량 중에서 조금 차별성을 가질 필요가 있음
    - 말하기
      - 조금만 질문이 달라져도 준비한 말들을 하지 못하는 경우가 많음
      - 외운 티가 난다.
        - 글로 작성한 자기소개를 읽어서 암기했기 때문
        - 말하기 연습을 해야 함
          1. 스크립트 작성 완료
          2. 꼭 말해야 하는 키워드를 선정
          3. 그 키워드만 보고 말함
          4. 말하는 과정에서 논리의 오류와 중복을 찾음
          5. 완성된 말하기 스크립트
          6. 영상으로 찍어서 - 비언어적인 표현

      - 기업에서 원하는 것에 대해서 먼저 말하고 자신이 원하는 메시지를 정할 것
      - 긴장해도 된다 = 적당한 긴장은 기업에 대한 갈망으로 보임
      - 생각이 먼저? 행동이 먼저?
